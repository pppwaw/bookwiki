# M0 M1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the BookWiki M0 project skeleton and M1 stub scheduler so a PDF-backed mini book can run from input to vault and SQLite without LLM calls.

**Architecture:** The initial implementation keeps all M1 behavior deterministic. A small graph runner exposes the same `build_graph` and `resume_or_start` entry points that later LangGraph code will keep, while nodes write the planned disk artifacts under each book directory. Stub agents return Pydantic models and the indexer builds a real SQLite database from generated Markdown.

**Tech Stack:** Python 3.12+, Pydantic, PyYAML, pytest, pytest-asyncio, Ruff, optional LangGraph/litellm/instructor/diskcache/tiktoken/MinerU runtime integrations.
